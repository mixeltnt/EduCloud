GUION COMPLETO PARA EXPONER EL PROYECTO EDUCLOUD
Landing Page con React, Vite y Bootstrap

============================================================
1. SALUDO E INTRODUCCION
============================================================

Buenos dias/tardes. Mi proyecto se llama EduCloud.

EduCloud es una landing page para una academia virtual de cursos online de programacion y diseno.
La idea principal del sitio es presentar la plataforma, mostrar sus cursos destacados, explicar sus beneficios,
mostrar al equipo docente, resolver preguntas frecuentes y permitir que un usuario se registre mediante un modal.

El proyecto fue desarrollado con React, Vite y Bootstrap.
Use React para separar la interfaz en componentes reutilizables, Vite para tener un entorno rapido de desarrollo
y Bootstrap para trabajar con una estructura responsiva usando clases como container, row, col, navbar, modal,
accordion, buttons y formularios.

============================================================
2. OBJETIVO DEL PROYECTO
============================================================

El objetivo fue crear una landing page clara, moderna y funcional.
No es solamente una pagina estatica, porque incluye interacciones hechas con React:

- Un menu responsive con boton hamburguesa.
- Un dropdown de cursos controlado con estado.
- Un carrusel de cursos con cambio automatico.
- Un acordeon de preguntas frecuentes.
- Un modal de registro funcional.
- Formularios con validacion.

Tambien se busco que el codigo quedara ordenado, separado por componentes y comentado para que sea facil de explicar.

============================================================
3. TECNOLOGIAS UTILIZADAS
============================================================

React:
Se uso para construir la interfaz por componentes. Cada seccion principal esta separada en su propio archivo.

Vite:
Se uso como herramienta para crear y ejecutar el proyecto. Permite levantar el entorno de desarrollo y compilar
la aplicacion de forma rapida.

Bootstrap:
Se uso para facilitar el diseno responsivo y algunos estilos base. Por ejemplo, se usan clases como:
container, row, col-lg-6, col-md-4, navbar, btn, badge, accordion, modal y form-control.

CSS personalizado:
Ademas de Bootstrap, se agregaron estilos propios en App.css y Navbar.css para darle identidad visual a EduCloud.

============================================================
4. ESTRUCTURA GENERAL DEL PROYECTO
============================================================

El componente principal es App.jsx.
Desde App.jsx se importan todas las secciones de la landing:

- Navbar
- Hero
- Features
- CoursesCarousel
- Instructors
- FAQAccordion
- Footer

La aplicacion se organiza de arriba hacia abajo como una landing page tradicional:

Primero aparece el Navbar, luego el contenido principal dentro de main, despues el Footer y finalmente el modal,
que solo se muestra cuando el usuario hace clic en los botones de registro.

============================================================
5. EXPLICACION DE APP.JSX
============================================================

En App.jsx se controla una parte importante de la aplicacion: el estado del modal.

El estado se llama showModal:

- Cuando showModal es false, el modal no se muestra.
- Cuando showModal es true, el modal aparece en pantalla.

Para manejarlo se usa useState:

const [showModal, setShowModal] = useState(false)

La funcion setShowModal(true) se pasa como prop al Navbar y al Hero mediante onRegisterClick.
Esto permite que dos botones distintos abran el mismo modal:

- El boton "Registrarme" del Navbar.
- El boton "Inscribirme gratis" del Hero.

Luego, el componente RegistrationModal recibe dos props:

- isOpen, que indica si el modal debe mostrarse.
- onClose, que permite cerrar el modal.

Esto demuestra comunicacion entre componentes usando props.

============================================================
6. EXPLICACION DEL NAVBAR
============================================================

El Navbar es la barra de navegacion superior.
Tiene el logo EduCloud, enlaces a secciones de la pagina, un dropdown de cursos y un boton para abrir el modal.

En este componente se usan dos estados:

isOpen:
Controla si el menu hamburguesa esta abierto o cerrado en pantallas pequenas.

coursesOpen:
Controla si el dropdown de cursos esta abierto o cerrado.

Aunque Bootstrap tiene comportamiento propio para navbars y dropdowns, en este proyecto la interactividad se controla
con React. Esto permite practicar el uso de useState y no depender del JavaScript de Bootstrap.

Tambien existe una funcion closeMenu.
Esta funcion cierra el menu hamburguesa y el dropdown cuando el usuario navega a una seccion.
Eso mejora la experiencia en pantallas pequenas.

Cuando el usuario hace clic en "Registrarme", primero se cierra el menu y luego se ejecuta onRegisterClick,
que viene desde App.jsx y abre el modal de registro.

============================================================
7. EXPLICACION DEL HERO
============================================================

El Hero es la primera seccion visible de la pagina.
Su funcion es presentar rapidamente la propuesta de EduCloud.

Incluye:

- El nombre del proyecto: EduCloud.
- Una descripcion breve de la academia.
- Un boton principal para inscribirse.
- Un boton secundario para ver los cursos.
- Un bloque visual que simula una vista de la plataforma.

El boton "Inscribirme gratis" ejecuta la funcion onRegisterClick, igual que el boton del Navbar.
Esto demuestra reutilizacion de una misma accion desde distintos componentes.

En la parte visual del Hero tambien se usan clases de Bootstrap como badge y progress para mostrar una clase en vivo
y una barra de progreso simulada.

============================================================
8. EXPLICACION DE FEATURES
============================================================

Features muestra los beneficios principales del metodo EduCloud.

En vez de escribir tres tarjetas manualmente, se creo un arreglo llamado features.
Cada objeto del arreglo tiene:

- icon
- title
- desc

Luego se usa map para recorrer el arreglo y generar una tarjeta por cada beneficio.

Esto es importante porque evita repetir codigo y hace que el componente sea mas facil de mantener.
Si quiero agregar otro beneficio, solo agrego otro objeto al arreglo.

Los beneficios que se muestran son:

- Rutas por nivel.
- Proyectos reales.
- Mentoria semanal.

============================================================
9. EXPLICACION DEL CARRUSEL DE CURSOS
============================================================

CoursesCarousel muestra los cursos destacados.

Los cursos estan guardados en un arreglo llamado courses.
Cada curso tiene:

- title
- category
- level
- text
- accent

El estado activeIndex indica que curso se esta mostrando actualmente.

Por ejemplo:

- Si activeIndex es 0, se muestra Frontend con React.
- Si activeIndex es 1, se muestra UX/UI desde cero.
- Si activeIndex es 2, se muestra Backend con Node.

El carrusel tiene tres formas de cambiar de curso:

- Boton anterior.
- Boton siguiente.
- Indicadores inferiores.

Tambien tiene cambio automatico usando useEffect.
Dentro de useEffect se crea un setInterval que cambia el curso cada 4.5 segundos.

La parte importante es que el intervalo se limpia con:

return () => clearInterval(timer)

Esto evita que el intervalo siga funcionando si el componente deja de mostrarse.
Es una buena practica en React cuando se usan temporizadores.

============================================================
10. EXPLICACION DE INSTRUCTORS
============================================================

Instructors muestra el equipo docente de EduCloud.

Igual que en Features, se usa un arreglo de datos llamado instructors.
Cada instructor tiene:

- name
- role
- initials
- bio

Luego se usa map para generar las tarjetas.

Esto permite que el contenido sea facil de modificar.
Por ejemplo, si quiero cambiar un instructor, no tengo que tocar toda la estructura HTML/JSX,
solo cambio el objeto correspondiente dentro del arreglo.

Tambien se usan clases de Bootstrap como row, col-md-4 y g-4 para que las tarjetas se acomoden de forma responsiva.

============================================================
11. EXPLICACION DEL ACORDEON DE PREGUNTAS FRECUENTES
============================================================

FAQAccordion muestra preguntas frecuentes sobre la academia.

Las preguntas estan guardadas en un arreglo llamado faqs.
Cada pregunta tiene:

- question
- answer

El estado openIndex indica cual pregunta esta abierta.

En el codigo actual, openIndex comienza en -1.
Eso significa que al cargar la pagina todas las preguntas aparecen cerradas.

Cuando el usuario hace clic en una pregunta:

- Si estaba cerrada, se abre.
- Si ya estaba abierta, se cierra.

Esto se hace comparando:

const isOpen = openIndex === index

Si isOpen es true, se agrega la clase show de Bootstrap para mostrar la respuesta.
Si isOpen es false, se agrega la clase collapsed al boton.

La logica del acordeon esta hecha con React, pero aprovecha las clases visuales de Bootstrap.

============================================================
12. EXPLICACION DEL MODAL DE REGISTRO
============================================================

RegistrationModal es el componente que muestra el formulario de registro.

Este componente recibe dos props:

isOpen:
Indica si el modal debe renderizarse o no.

onClose:
Funcion que viene desde App.jsx y permite cerrar el modal.

Si isOpen es false, el componente retorna null.
Eso significa que no se muestra nada en pantalla.

El formulario tiene tres campos:

- Nombre completo.
- Correo electronico.
- Curso de interes.

Para manejar el formulario se usan tres estados:

formData:
Guarda lo que escribe el usuario en los campos.

errors:
Guarda los mensajes de error cuando la validacion falla.

submitted:
Indica si el formulario ya fue enviado correctamente.

La funcion validate revisa:

- Que el nombre tenga al menos 3 caracteres.
- Que el correo tenga un formato valido.
- Que el usuario seleccione un curso.

La funcion handleChange actualiza el campo que el usuario esta modificando.
Para eso usa el atributo name del input o select.
Asi se puede usar una sola funcion para name, email e interest.

La funcion handleSubmit se ejecuta al enviar el formulario.
Primero usa event.preventDefault para evitar que la pagina se recargue.
Luego ejecuta validate.
Si hay errores, los muestra debajo de cada campo.
Si no hay errores, cambia submitted a true y muestra el mensaje de confirmacion.

Importante:
Este modal no guarda informacion en localStorage.
Solo valida los datos, muestra la confirmacion y limpia el formulario.

La funcion handleClose cierra el modal y deja sus estados limpios para la proxima vez que se abra.

============================================================
13. EXPLICACION DEL FOOTER
============================================================

El Footer es la parte final de la landing.

Incluye:

- El nombre EduCloud.
- Una descripcion breve.
- Enlaces internos a cursos, instructores y preguntas frecuentes.
- Informacion de contacto.
- Un formulario de newsletter.

El formulario del newsletter usa dos estados:

email:
Guarda el correo escrito por el usuario.

message:
Guarda el mensaje de respuesta, ya sea de error o de exito.

La funcion handleNewsletter evita que la pagina se recargue y valida el correo.
Si el correo no tiene formato valido, muestra:

"Ingresa un correo valido para suscribirte."

Si el correo es valido, muestra:

"Listo. Recibiras novedades y becas de EduCloud."

Despues limpia el campo del correo.

============================================================

La pagina es responsiva porque usa clases de Bootstrap y CSS personalizado.

Ejemplos:

18. PREGUNTAS POSIBLES Y RESPUESTAS
============================================================

Pregunta: Por que usaste componentes?
Respuesta: Porque React trabaja mejor separando la interfaz en partes reutilizables. Asi el codigo queda mas ordenado,
facil de leer y facil de mantener.

Pregunta: Cual es el componente principal?
Respuesta: App.jsx, porque organiza todas las secciones y controla el estado que abre y cierra el modal.

Pregunta: Como se abre el modal?
Respuesta: App.jsx tiene el estado showModal. Cuando el usuario hace clic en los botones de registro,
se ejecuta setShowModal(true), y eso hace que RegistrationModal se muestre.

Pregunta: Donde esta la validacion del formulario?
Respuesta: En RegistrationModal.jsx, dentro de la funcion validate. Tambien hay validacion en Footer.jsx
para el formulario del newsletter.

Pregunta: El formulario guarda datos?
Respuesta: No. En esta version el formulario no guarda datos en localStorage ni en una base de datos.
Solo valida, muestra una confirmacion y limpia el formulario.

Pregunta: Como funciona el carrusel?
Respuesta: Usa activeIndex para saber que curso mostrar. Los botones cambian ese indice y useEffect crea
un intervalo para cambiar automaticamente cada 4.5 segundos.

Pregunta: Para que sirve useEffect en este proyecto?
Respuesta: Se usa en el carrusel para ejecutar una accion despues de renderizar: iniciar el cambio automatico
de cursos. Tambien limpia el intervalo para evitar problemas de rendimiento.

Pregunta: Como funciona el acordeon?
Respuesta: Usa openIndex para saber que pregunta esta abierta. Si el usuario hace clic en una pregunta,
se actualiza openIndex y se agrega o quita la clase show de Bootstrap.

Pregunta: Por que no usaste el JavaScript de Bootstrap?
Respuesta: Porque queria controlar la interactividad con React. Bootstrap se usa principalmente para estilos
y estructura visual, mientras que React maneja los estados y eventos.

Pregunta: Como hiciste que la pagina sea responsiva?
Respuesta: Use clases de Bootstrap como container, row, col-md-4, col-lg-6 y navbar-expand-lg,
ademas de estilos personalizados en CSS.

Pregunta: Que parte fue la mas importante del proyecto?
Respuesta: La integracion entre componentes. Por ejemplo, App.jsx controla el modal y pasa funciones a Navbar y Hero
para que ambos puedan abrirlo.

Pregunta: Que mejorarias en una siguiente version?
Respuesta: Podria conectar el formulario a una API o base de datos, agregar imagenes reales de cursos,
mejorar accesibilidad y agregar pruebas automatizadas.

19. VERSION CORTA PARA DECIR DE MEMORIA
============================================================

Mi proyecto se llama EduCloud y es una landing page para una academia virtual de cursos online.
Fue creado con React, Vite y Bootstrap.

Use React para dividir la pagina en componentes como Navbar, Hero, Cursos, Instructores, Preguntas Frecuentes,
Footer y Modal de registro.

El componente App.jsx organiza toda la pagina y controla el estado del modal.
El Navbar y el Hero reciben una funcion por props para abrir ese modal.

Tambien usa useEffect en el carrusel para cambiar automaticamente de curso cada 4.5 segundos.

El modal valida nombre, correo y curso de interes. Si los datos estan correctos, muestra una confirmacion.
No guarda datos en localStorage.

La pagina tambien es responsiva gracias a Bootstrap y estilos propios.
En resumen, el proyecto demuestra componentes, props, estados, efectos, validaciones y renderizado dinamico en React.
